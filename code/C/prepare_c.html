  • 買本適合自己的參考書或找個教學網站

    因為C語言屬於編譯式語言，因此在撰寫時必須使用編譯器來進行編譯。何謂編譯式語言?一般坊間參考書大多推薦devC++下載連結，我認為整體而言是蠻不錯用，而像是Code:Blocks我覺得介面相較之下沒有devC++友善，看哪個順眼就用哪個吧

    下定決心每天練習

    每天練習才是學會寫程式的不二法門，就算每天只打一兩個程式出來，人有碰到電腦，其實就不太可能忘記那些語法，平常有空的時候可以把語法抄下來看一看，這樣比較容易記下來

    為何要有編譯式語言?

    簡單來說呢?我們所學的語法電腦其實看不懂，但是我們又不可能每個人都學會艱澀難懂的機器語言，所以高階語言因此就被發明出來了。

    編譯器的運作原理

    編譯器的原理大致上分為七個部分
  • 原始碼（source code）
    也就是我們所學的語法
  • 預處理器（preprocessor）
    在C語言中使用"#"開頭的指令就必須使用預處理器
  • 編譯器（compiler）
    也就是上述提到的devC++、Code:Blocks
  • 組合語言（assembler）
    現代大部分的開發者在進行軟體開發時不會使用此語言，但因為接近比起編譯語言更接近硬體，所以在一些要求時效性的程式中會使用組合語言開發，大致來說把編譯語言想成物理的公式，你其實不用了解背後複雜得運算過程你也可以算出答案，但在解某些難題時就必須了解過程
  • 目的碼（object code）
    內含可執行檔及函式庫，編譯過程的中間產物
  • 連接器（Linker）
    將所有文件連接在一起並產生可執行文件大致過程如圖
  • 執行檔（executables）
    也就是我們常看見的.exe
